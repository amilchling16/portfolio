ONBOARDING UX FLOW

BACKGROUND

In need of a user interface redesign, but without development resources to make the changes, I was tasked with creating a quick-win, low dev solution to provide new web and mobile app users with a successful first-time user experience. The solution needed to be completed within a single sprint.

My task was to transform known signup flow weak spots into an onboarding flow that would lead to increased adoption and long-term retention. This would be a big improvement for users, while buying us time before we created a UI overhaul.

NEW USER FLOW

I created a user flow to map every step of the process. This was a tool I used to ensure I was not only designing for the business goal that sparked the project (increase retention), but that I was also meeting user goals.

REQUIREMENTS

  • Create a streamlined signup process by simplifying the content architecture of the pricing page (project outlined here)
  • Provide a progress bar. Our job to be done is to increase productivity, we want a quick and effective process to uphold that objective
  • Gain insights into intended use cases by providing a one question survey
  • Spend greater focus on the connection between the web app and local app
  • Take new users through the real UI to connect them to the functionality, instead of leading them through abstract screens

WIREFRAMES

The decision was made to have the onboarding be a hands-on simulation of what the user would do on their own. This would allow us to shorten development time by recycling the existing UI. Tooltips were the chosen method for the post-signup tutorial. They would be easy for a developer to overlay, and if need be, we could use a third party service to create the tooltips without code.

I translated existing steps from the old onboarding to new layouts within the app UI.

FINAL INTERACTION DESIGN CONCEPT

Because this was a low-effort, high reward project, I was able to pass off placeholder concepts for the front-end developer to build from. I simply applied the new functions and copy over screenshots of the existing UI.

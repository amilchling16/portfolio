User Onboarding

Background

In need of a user interface redesign, but without development resources to make big changes, I was tasked with creating a quick-win, low dev solution to improve the first time user experience of a multi-platform app. The solution needed to be completed within a single sprint.

My specific task was to transform the existing signup flow's known weak spots into an onboarding flow that would lead to increased adoption and long-term retention. This would be a big improvement for users, while buying us time before we were able to release a total UI overhaul.

New User Flow

I created a high level user flow to map every step of the process. This was a tool I used to ensure I was not only designing for the business goal that sparked the project (increase user retention), but that I was also meeting the users' goals.

Requirements

  • Create a streamlined signup process by simplifying the content architecture of the pricing page
  • Provide a progress bar. Our primary job to be done was to increase productivity, so we wanted a quick and effective process to uphold that objective
  • Gain insights into intended use cases by providing a one question survey
  • Spend greater focus on the connection between the web app and local app
  • Take new users through the real UI in order to connect them to the real-life functionality, instead of leading them through abstract screens

Wireframes

The decision was made to have the onboarding be a hands-on walk through of what the user would do on their own post-onboarding. This would allow us to shorten development time by recycling the existing UI. Tooltips were the chosen method for the post-signup tutorial. They would be easy for a developer to overlay, and if need be, we could use a third party service to create the tooltips without any coding.

I rearranged and improved upon the flow of the old onboarding, connecting it into a new layout within the current UI.

Final Interaction Design

Because this was a low-effort, high reward project, I was able to pass off placeholder concepts for the front-end developer to build from. The only major update was the layout redesign of the Pricing page to improve scannability and more strongly differentiate the value of paid plan options. For the rest, I simply applied the new functions and copy over screenshots of the existing UI.
